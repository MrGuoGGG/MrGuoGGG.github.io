第八章 继承、抽象类、接口和枚举

更新于2022.10.5早

8.1 类的继承

  • 被继承的类称为父类，由继承而得到的类称为子类。
  • 一个父类可同时有多个子类，但一个子类只能有一个父类。
  • 父类实际上是所有子类的公共成员的集合，每一个子类是父类的特殊化，是对公共成员变量和方法在功能、内涵上的扩展和延伸。
  • 子类继承父类可访问的成员变量和方法，同时可以修改父类的成员变量或重写父类的方法，也可以添加新的成员变量和方法。
  • 在Java语言中有一个名为Object的类，所有的类都是直接或间接继承该类得到的。

8.1.1 子类的创建与继承关系的uml图

####1. 子类的构建方法

子类的每个对象也是其父类的对象，反之则不然。

子类继承父类的UML图用带空心三角箭头的实线描述，实线的起始是子类的UML图，终点是父类的UML图。

  • 在Java语言的继承中，执行子类的构造方法之前，会先调用父类中没有参数的构造方法，其目的是帮助继承自弗雷德成员做初始化操作。
  • 一般情况下，最好为每个类提供一个无参构造方法。

####2. 调用父类中特定的构造方法

  • 使用super(参数列表)语句。super()可以重载，即super会根据参数的类型和个数，执行父类相应的构造函数。
  • 调用父类方法的super语句必须写在子类构造函数的第一行，否则编译时将出现错误。
  • Java程序在执行子类的构造函数之前，如果没有用super来调用父类中特定的构造函数，则编译器就会把super()作为构造函数的第一条语句，调用父类中没有参数的构造函数。
  • super()与this()功能相似，但super是从子类的构造方法调用父类的构造方法，this则是在同一个类内调用其他的构造方法。
  • super()与this()必须放在构造函数内的第一行，因此无法共存于同一个构造函数内。

8.1.2 在子类中访问父类的成员

在子类中使用super不但可以访问父类中的构造函数，还可以访问父类的成员变量和成员函数，但super不能访问在子类中添加的成员。

由于在子类中不能继承父类中的private成员，所以无法在子类中访问父类的这种成员，但protected可以访问。

8.1.3 覆盖

覆盖与重载相似，都是Java多态的技巧。
覆盖是指在子类中，定义名称、参数个数与类型均与父类中完全相同的函数，用来重写父类里同名函数的功能。

####1. 覆盖父类的函数
子类继承父类中所有可被访问的成员函数时，如果子类的方法头与父类中的方法头完全相同，则不能继承，此时子类的方法覆盖父类的方法。

###2.用父类的对象访问子类的成员
当指向子类对象的引用赋值给父类的引用变量时，只能访问子类对象在父类中定义的那些部分。

  • 创建父类类型的变量指向子类对象，称为向上转型，是安全的；同时也有向下转型。当在程序中使用向下转型技术时，必须使用显式类型转换。

8.1.4 不可被继承的成员与最终类

在默认情况下，所有的成员变量和成员函数都可以被覆盖，如果父类的成员不希望被子类的成员所覆盖，则可以将他们声明为final。
用final修饰的成员变量称为最终变量
用final修饰的成员方法称为最终方法，二者都不可被覆盖。
如果一个类被final所修饰，则说明这个类不能够在被其他类所继承，称为最终类。

  • 所有已被private修饰符限定为私有的方法，以及所有包含在final类中的方法，都被默认为是final的。

8.1.5 Object类

|常用方法|功能说明|
|public boolean equals(···)|判断两个对象变量所指向的是否为同一个对象|
|public String toString|将调用toString()方法的对象转换为字符串|
|public final Class getClass()|返回运行时对象所属的类|
|protected Object clone()|返回调用该方法的对象的一个副本|
